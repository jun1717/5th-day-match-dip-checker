# 運用マニュアル — 迷わないための使い方

このドキュメントは「毎日・売買時・週次・月次・四半期に何をするか」を1冊で決める運用手順書です。
機能の仕様は [README.md](../README.md)、ルール変更の検証記録は [docs/backtest-results/](backtest-results/) を参照。

## 大原則(これだけ覚える)

1. **判断は1日1回、大引け後の確定データで行い、翌朝執行する。** 場中は見ない。
2. **あなたの仕事は3つだけ**: ①夜に候補を見て翌朝の注文を準備する ②約定したら `executions.csv` に1行書く ③月次で振り返る。それ以外(価格取得・スコア計算・履歴保存)は全部自動。
3. **ルールは思いつきで変えない。** 変えたくなったらバックテストでA/B比較 → 採用基準を満たしたら変更 → `docs/backtest-results/` に記録。この順番以外で `rules.json` を触らない。

## 1. 毎日の運用(平日夜 16:40以降・10分)

16:30 JST に確定日足でその日の判定がスナップショットされます。**それ以降に**サイトのトップページを開く。

### 手順

0. **警告バナーが出ていないか見る**(トップページ最上部)。**赤(確定日足ではない)が出ていたら、この画面で注文を組まない**。16:30の自動更新が遅れているだけのことが多いので、時間をおいて再読み込みして消えてから判断する。黄(履歴保存停止)は当日の判断には影響しない(週末にGitHub Actionsのログを確認)。
1. **地合いカードを見る**(トップページ上部)。
   - OK/NGを確認。**NGでも候補を機械的に捨てない**(2026-07検証でexclude不採用: NG日はむしろ深押しの好機だった)。ただしNGの日は「新規は普段の半分まで」など自分の裁量ルールをメモしておくと迷わない。
2. **買い候補一覧を見る。** 買い候補(buy_candidate)だけでよい。監視・見送りは開かない(時間の無駄)。**候補ゼロの日は何もしない日**。それがこのツールの正常動作です。
3. 買い候補があれば銘柄詳細ページで以下を確認:
   - **reasonsのフラグ**: `stop_too_tight` / `volume_not_dry` / `market_regime_weak` は**注意表示であって除外理由ではない**(いずれも検証済みでexclude不採用)。同じ日に候補が複数あってどれか選ぶなら、フラグなしを優先する程度でよい。
   - **決算日**: `earnings.csv` に入れた銘柄は `earnings_soon` で自動降格される。**まだ入れていない銘柄は、3営業日以内に決算がないか手動確認**(唯一の手動チェック項目)。
4. **翌朝の注文を組む**(下の「注文の組み方」)。
5. 終わり。チャートを眺めて悩む時間は成績に寄与しません。

### 注文の組み方(買い候補1銘柄につき)

| 項目 | 値 | 備考 |
|---|---|---|
| 買い指値 | 買い基準価格(5日線×1.005) | 買い上限(×1.015)を超えては**絶対に追わない**。届かなければ縁がなかった |
| 損切りライン | **今日(シグナル日)の安値** | 「エントリー日の前日の安値」がドクトリン。夜に見て翌朝買うなら今日の安値。**画面の「損切りライン」列がこの値**。**エントリー後は動かさない(固定)** |
| 株数 | 画面の「株数」欄をそのまま使う | `1.2万円 ÷ (買い指値 − 今日の安値)` を100株単位で切り捨てた値が表示済み(シグナル日の安値基準)。`maxPositionYen` 超えは0株=見送りと表示される |
| 損切り注文 | 買いと同時に逆指値をセット | 「約定したら置く」ではなく最初から。裁量を挟まないため |

**入らない条件**(バックテストも同じ仮定):
- 買い指値 ≤ 損切りライン(そもそも成立しない)
- 翌朝の寄りが損切りライン以下で始まった(ギャップダウン。シナリオ崩壊済み)

### 保有中の出口ルール(エントリー時に決めて固定)

エントリー時のテーマ資金スコアで出口モードが決まり、**途中で切り替えない**:

- **テーマスコア < 90(通常モード)**: 第1利確=直近20日高値に売り指値。損切り逆指値と両方置いて放置。
- **テーマスコア ≥ 90(トレンド継続モード)**: 利確指値は置かない。**終値が5日線を割った日**に売る(引けか翌朝寄り)。夜のルーチンで保有銘柄の終値と5日線を見るだけ。
- どちらでも: 損切りは固定。**30営業日**経っても決着しなければ手仕舞い(通常モードのみ。トレンド継続モードは5日線割れまで)。

### 朝(任意・3分)

寄り前に注文の入り忘れがないか確認する程度。判断は前夜に終わっている。

### 場中は見ない

20分毎の自動更新は「どうしても確認したい時」のための参考表示。場中の出来高比は偽陽性方向に出る(README明記)し、場中に思いついたエントリーは月次レビューで `same_day_signal`(場中判断)の逸脱フラグが付きます。**サイトを場中に開かないのが一番簡単な規律**です。

## 2. 売買したら必ずやること(その日のうちに・1分)

`data/trades/executions.csv` に1行追記。買いも売りも、部分約定も1行ずつ:

```csv
executedAt,code,side,price,shares,memo
2026-07-15,5803,buy,6210,300,寄り後に指値で約定
```

これはツール全体で**唯一あなたにしか入力できないデータ**で、これが無いと月次振り返り(勝てているか・ルール通りか)が丸ごと機能しません。「あとでまとめて」は必ず漏れるので約定当日に。

> リポジトリがpublicの場合、コミットすると売買記録が公開されます。避けるなら `data/trades/` を `.gitignore` に入れてローカル管理(README参照)。

## 3. 週次(週末どこかで・10分)

1. **`git pull`** — GitHub Actionsが毎日コミットしているシグナル履歴(`data/history/`)を手元に同期。
2. **ウォッチリストのメンテ** — テーマの主役交代や新テーマの兆しがあれば `data/watchlist.csv` を編集。新規銘柄探しには jp-momentum-monitor を使う:
   ```bash
   cd ~/Desktop/work/jp-momentum-monitor
   uv run momentum-monitor rank
   ```
   上位に出た銘柄のうち、既存テーマに合うものを watchlist.csv に追加(`theme` と `is_leader` を忘れずに)。**ここは唯一の裁量ポイント**なので週1回に制限する(毎日いじるとテーマスコアの母集団が動いて履歴の意味が薄れる)。
3. watchlist を変えたら、新銘柄の決算日も `earnings.csv` に追加。

## 4. 月次(月初の週末・30分) — ここで「勝てているか」を判定する

Claude Code でプロジェクトを開いて:

```
/monthly-review 2026-07
```

と打つだけ(スキル [.claude/skills/monthly-review/](../.claude/skills/monthly-review/SKILL.md) が下記のコマンド実行から解釈・レポートまで行う)。手動でやる場合:

```bash
git pull
python3 scripts/fetch_prices.py          # ローカルの価格データを最新化
npm run review:trades -- --month 2026-07 # 先月分
npm run analyze:signals
```

### review:trades で見るもの(優先順)

1. **`holding_below_stop`(損切りライン割れを保有中)** — 出ていたら最優先。翌朝どうするか、その場で決める。
2. **逸脱フラグ集計** — `not_buy_candidate`(ルール外エントリー)・`chase_entry`(買い上限超え)・`late_stop`(損切り遅れ)が月に何回あったか。**ゼロが目標**。
3. **実成績 vs 仮想成績(ツールに完全に従った場合)の差** — 実際の株数で計算されるので、差は純粋に執行の質。
   - 実成績 < 仮想成績 → **執行の問題**。ルールは触らず、注文を機械的に置く運用(逆指値の事前セット等)を徹底する。
   - 仮想成績そのものが悪い → **ルールの問題の可能性**。ただし1ヶ月では判断しない(下記)。

### analyze:signals で見るもの

status別(buy_candidate / watch)のフォワードリターンと約定シミュレーション成績。**シグナル履歴は2026年7月から蓄積開始**なので、意味のある数字になるのは2〜3ヶ月後。それまでは「溜まっているか」の確認だけでよい。判断材料が欲しい間は2年バックテストの数字(現行設定で期待値R 0.68・PF 1.79・勝率23%。[2026-07-max-position.md](backtest-results/2026-07-max-position.md))を参照——**勝率2割強で期待値を出す設計**なので、数連敗は正常の範囲。ここで閾値をいじりたくなるのを我慢するのがこの仕組みの肝です。

## 5. 四半期(決算シーズン前: 1月・4月・7月・10月・30分)

## 6. ルールを変えたくなったら(不定期)

衝動で `rules.json` を触らず、Claude Code で:

```
/backtest-ab volume-filterをexcludeにしたい
```

のように変更案を渡す(スキル [.claude/skills/backtest-ab/](../.claude/skills/backtest-ab/SKILL.md) が蒸し返しチェック→A/B実行→基準判定→`docs/backtest-results/` への記録まで行う)。手動でやる場合:

```bash
python3 scripts/fetch_prices.py --period 2y --output data/prices_backtest.csv --no-intraday --auto-adjust
npm run backtest -- <変更オプション> --out data/backtest/<実験名>
npm run backtest -- --out data/backtest/baseline   # 現行設定
```

- **採用基準**(過去の検証と同じ): ベースライン比で期待値Rが改善し、かつ約定数が60%以上残る。
- 採用したら `docs/backtest-results/YYYY-MM-<テーマ>.md` に記録(過去3件の形式に倣う)。
- バックテストの絶対値は楽観的(ウォッチリスト自体が後知恵選択)。**構成間の相対比較にだけ**使う。

### 検証済みで確定していること(蒸し返さない)

| 項目 | 結論 | 記録 |
|---|---|---|
| 建玉方式 | `risk` 採用(期待値R 0.546→0.619) | [2026-07-atr-volume.md](backtest-results/2026-07-atr-volume.md) |
| ATR損切り品質・出来高ドライアップ | `flag` 維持(exclude は成績悪化。仮説と逆の傾向) | 同上 |
| 地合いフィルター | `flag` 維持(exclude で期待値R 0.67→0.49) | [2026-07-market-filter.md](backtest-results/2026-07-market-filter.md) |
| テーマスコア方式 | `binary` 維持 | [2026-07-theme-scoring.md](backtest-results/2026-07-theme-scoring.md) |
| 決算フィルター | `exclude` 採用(性能仮説ではなくリスク上限の不変条件) | README |
| 1銘柄投入上限 | `maxPositionYen: 1000000`(期待値R維持・最大DD -31%) | [2026-07-max-position.md](backtest-results/2026-07-max-position.md) |

### 予約済みの再検証(忘れないこと)

- **地合いフィルター**: 検証期間が強気相場のみ。**次の本格的なベア相場を経験したら必ず再検証**(2026-07-market-filter.md に明記)。
- **出来高ドライアップ**: コホートは「出来高増の押し目ほど良い」という仮説と逆の傾向。実運用のシグナル履歴が溜まったら `analyze:signals` で再確認。
- **決算フィルター**: `earnings.csv` の履歴が2〜3四半期溜まったら `--earnings-filter off` とA/B。

### 夜の判断フロー(1枚版)

```
サイトを開く(16:40以降)
 ├ 赤い警告バナー → 判断しない。再読み込みして消えるのを待つ
 ├ 買い候補ゼロ → 何もしない(正常)。保有銘柄の出口だけ確認して終了
 └ 買い候補あり
    ├ 3営業日以内に決算? ─ はい → 見送り(earnings.csv未登録銘柄のみ手動確認)
    ├ 翌朝の注文をセット(いずれも画面の値をそのまま使う):
    │   買い指値 = 買い基準価格(上限超えは追わない)
    │   損切り  = 損切りライン(今日の安値。逆指値を同時セット・以後動かさない)
    │   株数    = 株数欄(1.2万円 ÷ (指値 − 今日の安値) の100株単位切り捨て。
    │            0株=見送り)
    └ 出口モードをメモ:
        テーマ<90 → 20日高値に利確指値 / テーマ≥90 → 5日線終値割れまで保有
翌朝: 寄りが損切り以下なら入らない。指値に届かなければ諦める
```
