# 検証記録: テーマスコア連続値化 binary vs continuous (2026-07-12)

`docs/plan-theme-scoring-and-trade-review.md` のM3(A/B検証と切替判断)の実施記録。

**結論: continuous への切替は見送り。`themeScoringMode: "binary"` を維持する。**

## 検証環境

- 価格データ: `data/prices_backtest.csv`(2年分・分割調整済み・確定日足。2026-07-10まで)
- 対象期間: 2025-07-23 〜 2026-06-12(216営業日)、対象status: buy_candidate、stopMode: prev-day、maxHoldDays: 30
- rulesHash: `8fc10ada0490`(sizing=risk / 両品質フィルター=flag / themeMomentumBlend5d=0.6 / themeMomentum5dRange=0.05 / themeMomentum20dRange=0.1)
- `--theme-scoring` のCLIオーバーライドのみで比較(rules.jsonは両構成で共通)

## 採用基準の判定(plan B-4)

| # | 基準 | 結果 | 判定 |
|---|---|---|---|
| 1 | 境界安定性: flipsPer100Daysが減少 | 31.32 → 32.24(微増) | **不合格** |
| 2 | 期待値Rがbinary比 −0.05R以内 | 0.62 → 0.38(−0.24R) | **不合格** |
| 3 | シグナル数がbinary比 70〜130% | 101% | 合格 |
| 4 | trend_follow比率が激増しない | 92%→37%に**激減**(下記) | 要注意 |
| 5 | テーマスコア帯でfwd5/fwd20が単調改善 | buy側は単調、watch側は80-89帯に凹み | 部分合格 |

2基準(しかも主目的の安定性を含む)が不合格のため、planの判断ルールどおり**binaryを維持**する。

## 何が起きたか(exitMode別・テーマスコア帯別)

binary(トレード側):

| 区分 | 約定/シグナル | 勝率 | 期待値R | 累積損益 |
|---|---|---|---|---|
| trend_follow_exit(スコア90-100) | 251/531 | 21.1% | 0.64 | 103.9万円 |
| target_exit(スコア80-89) | 20/44 | 35.0% | 0.30 | 7.5万円 |

continuous(トレード側):

| 区分 | 約定/シグナル | 勝率 | 期待値R | 累積損益 |
|---|---|---|---|---|
| trend_follow_exit(スコア90-100) | 94/215 | 18.1% | **0.77** | 27.1万円 |
| target_exit(スコア80-89) | 177/368 | 26.0% | 0.17 | 36.5万円 |

1. **成績劣化の正体は閾値80の較正ズレであってスコアの質ではない**。binaryでは「スコア80以上」の実体はほぼ100点テーマだけ(シグナルの92%が90-100帯)だった。continuousでは中位テーマ(相対強度+維持率だけで80に届く)が80-89帯に大量流入し(44→368シグナル)、この帯の期待値Rは0.17しかないため全体を薄めた。
2. 一方で**continuousの90-100帯はbinaryの90-100帯より質が高い**(期待値R 0.77 vs 0.64。コホートでもfwd20 +7.7% vs +6.7%)。連続スコアの上位選別自体は機能している。
3. **binaryの成績の主要ドライバーは「テーマスコア100のみtrend_follow」という実質的な仕様**だった(平均勝ちRが6.52 vs 5.01)。continuousでは90-100が連続になりtrend_follow対象が広がった結果、逆にtrend_follow比率が激減した(80台に沈むテーマが増えたため)。
4. 安定性: 日次スコアの暴れはcontinuousで大きく減った(18.5→13.3点)が、statusフリップは減らなかった。binaryのスコアは大きく跳ぶが同一status内(0/20/30/50はどれもweak)での跳びが多いのに対し、continuousは閾値60/80の近傍を滑らかに漂うため境界を跨ぐ回数はむしろ増える。**フリップ低減には連続値化ではなくヒステリシス(status変更に非対称閾値)が本命**と分かった。
5. 較正プローブ: モメンタム正規化幅を±8%/±16%に広げても改善なし(期待値R 0.40、フリップ32.28)。この軸の調整では救えない。

## 将来の再検証に向けた較正案(優先順)

1. **ヒステリシス導入とセットで再検証**(plan 9の将来拡張)。フリップ低減という本来の目的にはこちらが筋が良い。
2. continuous採用時は `themeBuyScoreThreshold` を80→90近辺へ引き上げる案: 90-100帯単体(0.77R)はbinary全体(0.62R)を上回るが、シグナル数が約37%(215件)に減り累積損益は27万円と大幅減。トレード数維持の観点で現時点では不利。
3. 相対強度・維持率・モメンタムの配点バランス見直し(80-89帯のfloodは相対強度30点+維持率50点だけでほぼ80に届く構造のため。モメンタムの配点を厚くする等)。
4. watch側コホートの80-89帯の凹み(5日後プラス率53.1%)の原因調査。

## 注意事項(結果の解釈)

- ウォッチリスト自体が後知恵選択のため絶対値は楽観的。判断はすべて構成間の相対比較に基づく。
- 期間が約10.5ヶ月(216営業日)の単一の強気局面。トレンドフォロー優位(=90-100帯偏重が有利)は地合い依存の可能性があり、下落局面を含むデータで結論が変わりうる。
- 同日stop/TP同時成立は損切り扱い(保守的)、手数料・スリッページは0。
- 実装(scoring.ts の continuous 経路・バックテストの帯別集計・安定性指標)は本検証のために本体に残してある。`--theme-scoring continuous` でいつでも再検証できる。

## 再現コマンド

```bash
python3 scripts/fetch_prices.py --period 2y --output data/prices_backtest.csv --no-intraday --auto-adjust
npm run backtest -- --theme-scoring binary     --out data/backtest/theme-binary
npm run backtest -- --theme-scoring continuous --out data/backtest/theme-continuous
# 較正プローブは config/rules.json の themeMomentum5dRange=0.08 / themeMomentum20dRange=0.16 に一時変更して実行
```
